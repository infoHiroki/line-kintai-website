# 警備業向けLINE勤怠管理サービス - Appleスタイルウェブサイト技術仕様書

## 1. プロジェクト概要

### 1.1 目的とターゲット
- **目的**: ITリテラシーが低い警備業界従事者向けにLINE勤怠管理サービスを訴求する営業用ウェブサイト制作
- **ターゲット**: 
  - 警備会社の経営者・管理者（主な意思決定者）
  - 現場管理者（実務担当者）
  - 経理担当者（給与計算担当者）
  - ITリテラシーが低い年配の警備員（エンドユーザー）
- **サービスコンセプト**: 「簡単・正確・迅速」な勤怠管理で現場と経理の連携強化

### 1.2 サービスの特徴と差別化ポイント
- **特徴**: 
  - LINEを活用した直感的な勤怠管理
  - 出退勤をLINEメッセージで簡単報告
  - スプレッドシートへの自動出力
  - 複数現場の一元管理
  - リアルタイム勤怠状況確認
  - 専用機器不要で即日導入可能
- **差別化ポイント**: 
  - LINE連携
  - スプレッドシート連携
  - 警備業特化
  - 導入の容易さ
  - 地域密着サポート

### 1.3 現状の課題
- 複数現場の警備員の勤怠管理が煩雑
- 紙ベースでの非効率な管理
- ITリテラシーの低い警備員への新システム導入の難しさ
- 経理の締め日に間に合わないケースの発生
- 警備日報と実際の勤務時間の差異
- 警備員と経理担当者の勤務時間認識の差
- 電話中心のコミュニケーションによる伝達の遅れ

## 2. ウェブサイトデザイン要件

### 2.1 全体コンセプト
- **デザインアプローチ**: 
  - Appleのウェブサイトを参考にした洗練されたデザイン
  - 視覚的要素を優先し、文字情報を最小限に抑えたレイアウト
  - スクロールに連動したストーリーテリング形式の情報提示
  - 直感的に理解できる視覚表現（特にITリテラシーが低いユーザー向け）

### 2.2 ビジュアル設計
- **カラースキーム**: 
  - メインカラー: 信頼感を与えるブルー系
  - アクセントカラー: 警備業を想起させるイエロー系
  - 意味による色分け（機能:青、メリット:緑、導入手順:オレンジ）
- **タイポグラフィ**: 
  - 見出し: 太字のサンセリフ（高齢者にも読みやすいサイズ）
  - 本文: 読みやすさを重視したサンセリフフォント
  - フォントサイズ: モバイルでも読みやすい大きめの設定
- **画像・図解**: 
  - 実際のLINE画面のスクリーンショット
  - 簡潔なアイコンとイラスト
  - Before/After比較画像
  - ステップバイステップの操作ガイド画像
  - 情報フロー図とプロセス図解

### 2.3 レイアウトと構成
- **ページ構成**: 
  - ワンページ構成（長尺スクロールページ）
  - セクションごとにスティッキー要素を活用
  - 明確なセクション区分（サービス概要→課題→解決策→導入手順→事例）
- **レスポンシブ対応**: 
  - スマートフォン/タブレット/デスクトップすべてで最適表示
  - タッチデバイス向け操作性の最適化
  - デバイスごとのコンテンツ表示優先順位の調整
- **情報密度**: 
  - 1セクションあたり3-5ポイントに情報を制限
  - 十分な余白を確保したゆとりあるデザイン
  - 情報の階層化（重要なポイントを視覚的に強調）

## 3. スクロールアニメーション要件

### 3.1 スクロール連動アニメーション
- **基本要件**: 
  - スクロール位置と完全同期したアニメーション
  - スクロール速度に応じたアニメーション進行速度の調整
  - スクロール停止時のアニメーション即時停止
- **ストーリーテリング**: 
  - 課題→解決→効果の流れをスクロールで展開
  - 異なるユーザー（警備員・管理者・経理）の視点を順に表示
  - 導入前後の比較を劇的に演出するトランジション

### 3.2 パララックス効果
- **導入シーン**: 
  - 複数レイヤーの要素が異なる速度でスクロール
  - 前景要素（LINEアプリUI）と後景要素（業務フロー）の立体的表現
  - 奥行き感を利用した情報の流れ（現場→管理者→経理）の視覚化
- **表現技法**: 
  - Z軸上の移動で重要度の違いを表現
  - 注目させたい要素の前面への浮き出し効果
  - スクロール方向に対する逆方向移動による視覚的インパクト

### 3.3 スティッキーセクション
- **活用方法**: 
  - サービスの主要機能解説時に画面左側に固定表示
  - 固定された機能説明に対し、右側に具体的なユースケースを流動表示
  - 比較表示（Before/After）で「導入前」を固定し「導入後」をスクロール表示
- **表現効果**: 
  - LINE勤怠報告の操作フローをスティッキー表示
  - 管理画面の機能を固定しながら各機能の詳細をスクロールで展開
  - 固定期間終了後の自然なスクロールフローへの復帰

### 3.4 警備業向け特化アニメーション
- **情報フロー可視化**: 
  - 現場からLINEを通じて経理までの情報の流れを視覚化するアニメーション
  - 紙ベースの勤怠管理が電子化されていく変化を表現
  - リアルタイムデータ連携のスピード感を表現するアニメーション
- **ユーザー別視点切替**: 
  - 警備員視点→管理者視点→経理視点を切り替えるトランジション
  - 各ユーザーの問題解決を段階的に表示するアニメーション

### 5.3 Intersection Observer の実装
```javascript
const observerOptions = {
  root: null, // ビューポートを基準
  rootMargin: '0px',
  threshold: buildThresholdList() // 複数のしきい値を設定
};

const observer = new IntersectionObserver((entries) => {
  entries.forEach(entry => {
    // entry.intersectionRatio に基づいてアニメーションの進行度を調整
    const progress = Math.min(1, Math.max(0, entry.intersectionRatio));
    
    // 警備業サービス紹介のアニメーション更新
    updateServiceAnimation(entry.target, progress);
  });
}, observerOptions);

// 0から1までの細かいしきい値リストを生成
function buildThresholdList() {
  let thresholds = [];
  let numSteps = 100;
  
  for (let i = 0; i <= numSteps; i++) {
    thresholds.push(i / numSteps);
  }
  
  return thresholds;
}

// 監視対象の要素を登録（警備業サービスの各セクション）
document.querySelectorAll('.service-section').forEach(element => {
  observer.observe(element);
});
```
